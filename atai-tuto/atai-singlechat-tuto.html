Description

Nota bene : le code qui s'affiche dans cette page est une demo fonctionnelle à visée strictement illustrative (les liens ne dirigent pas forcément aux endroits concernés). Le code fourni plus bas, à donner aux membres, est la version vierge à compléter.

Nota Bene : le code est hébergé via Github Pages pour une maintenance plus efficace à l'avenir, et cette page est réservée au Staff de JM (un genre de tuto que je peux alimenter au fur et à mesure pour corriger des trucs).

Je ne propose dans cette page qu'un descriptif du code à usage du Staff. Le code final est très simple à compléter, mais je préfère m'étendre ici de façon exhaustive pour couvrir tous les aspects du code et prévoir d'éventuelles questions. Je vous laisse gérer la façon dont vous le présenterez aux membres (c'est-à-dire, de façon plus concise !!) ^^

Description

Le code propose une interface Web classique d'affichage de messages type "mail" pour le Singlechat de l'application ATAI. Il se décline en 4 versions : pour un membre qui répond à un questionnaire en public sur le Singlechat, pour l'Incontestable qui publie un questionnaire (+ un message facultatif) en public sur le Singlechat, pour l'Incontestable qui publie une réponse anonyme à un questionnaire en public sur le Singlechat, et pour le gouvernement Japonais qui publie un message quelconque en public sur le Singlechat.

  • Nota bene : le logo et le titre + icône dU Singlchat sont injectés directement par le code CSS afin de ne pas surcharger le HTML.
  • L'entête du message contient les éléments personnalisables à compléter suivants : l'avatar de l'expéditeur, le nom complet de l'expéditeur, le lien vers le profil ATAI de l'expéditeur sur le fow, la mention du carcatère public de la publication en lien-vide, et la date de publication du message.
  • Le corps du message, dans la partie rose, contient le titre/objet du message et le contenu du message. Le titre et les paragraphes sont à compléter. Chaque paragraphe doit être inclus entre balises <p></p>, sauf s'il s'agit des questions/réponses au questionnaire, auquel cas il faut respecter la mise en forme sous liste à puces donnée par <ul><li></li></ul>.

Code

Voici le code HTML des 4 versions de messages Singlechat. Il suffit de sélectionner l'une des 4 versions souhaitée dans le code puis à la copier-coller puis à la compléter : https://raw.githubusercontent.com/manonstv/JM-LS/refs/heads/main/atai-singlechat/atai-singlechat.html.

Vous accéderez au code "brut" des 4 parties HTML, qu'il suffit de sélectionner puis de copier-coller entre balises [code][/code] sur le forum pour le proposer aux membres.

Consignes

  • Les parties du code à compléter sont signalées par un commentaire en début de ligne. Pour les URL d'images et les liens à compléter, les indications sont signalées en lettres capitales et _underscores_ dans les balises d'images et de liens concernées.
  • Ne pas ajouter d'espaces ou de sauts de lignes. Ils sont inclus et gérés par défaut par le code CSS.
  • Tout ajout doit se faire par strict retour à la ligne en intégrant les parties de codes fournies par ce tutoriel.
  • Ne supprimer aucun caractère ou aucun bout de code à l'exception de ceux expliqués dans ce tuto, car ils ont tous une utilité.
  • Encore une fois, la partie HTML ne doit contenir aucun saut de ligne. Il faut préciser aux membres que le code ne contient AUCUNE erreur. Ils ne doivent donc pas retirer le moindre caractère du code, y compris les commentaires HTML cassés, sous peine de compromettre l'intégrité du rendu.

Astuce : les injections automatiques de sauts de lignes (<br>) intempestifs par le script de forumactif sont contournées grâce à une astuce donnée par Yuna : afin de rendre le code HTML + lisible, les retours à la ligne sont simulés par le "cassage" de commentaires HTML (<!-- -->), permettant des retours à la ligne dans le code source afin de le rendre moins dense et plus facilement personnalisable, le tout sans conséquence dans le rendu.

Personnalisation

  • Rigoureusement identique au code pour la messagerie ATAI.
